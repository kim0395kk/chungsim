# 🎯 정부 민원 AI 시스템 개선 프롬프트 (Claude Code용)

## 📋 프로젝트 개요
이 프로젝트는 Streamlit 기반의 정부 민원 처리 AI 시스템입니다. 주요 기능은:
- AI 생성 민원의 환각(hallucination) 탐지
- 업무 지시 자동 분석 및 초안 생성
- 법령 검색 및 절차 가이드 제공
- 문서 자동 생성 (DOCX, PDF)
- 후속 질문 처리

## 🎁 개선 목표
**사용자 만족도를 극대화하는 코드 개선**을 목표로 합니다. 다음 관점에서 접근해주세요:

### 1. 🚀 성능 최적화 (체감 속도 향상)
- **LLM 호출 최적화**: 불필요한 API 호출 제거, 배치 처리, 병렬화
- **캐싱 전략 강화**: `@st.cache_data`, `@st.cache_resource` 적극 활용
- **DB 쿼리 최적화**: Supabase 쿼리 개선, 인덱싱 제안
- **스트리밍 응답**: 긴 응답은 스트리밍으로 점진적 표시
- **프리로딩**: 사용자가 다음에 필요할 데이터 미리 로드

**중점**: 사용자가 "빠르다"고 느끼는 것이 목표입니다.

### 2. 🎨 UX 개선 (사용자 경험)
- **로딩 시간 체감 개선**: 
  - 프로그레스바, 단계별 안내 메시지
  - "AI가 법령을 검색 중입니다..." 같은 구체적 피드백
- **에러 처리 개선**:
  - 친절한 에러 메시지
  - 복구 방법 제시
  - 부분 성공 처리 (일부 실패해도 진행)
- **입력 검증 강화**: 사용자가 잘못된 입력 하기 전에 안내
- **키보드 단축키**: 자주 쓰는 기능에 단축키 추가
- **자동 저장**: 입력 내용 자동 저장 (브라우저 새로고침 대비)

### 3. 🧠 지능형 기능 추가
- **스마트 제안**: 
  - 과거 유사 케이스 자동 추천
  - 자주 쓰는 문구 템플릿 제안
- **학습 기능**:
  - 사용자 피드백 기반 개선
  - 부서별 선호 응답 스타일 학습
- **프로액티브 알림**: 
  - "이 법령은 최근 개정되었습니다" 알림
  - "비슷한 민원이 3건 있습니다" 제안

### 4. 🛡️ 안정성 강화
- **환각 탐지 정확도 향상**:
  - 더 많은 패턴 규칙 추가
  - 법령 DB와 실시간 대조
  - 확신도 기반 단계별 경고
- **복원력 증대**:
  - API 실패 시 폴백 전략
  - 부분 데이터로도 작동
  - 타임아웃 처리 개선

### 5. 🎯 핵심 사용 시나리오 최적화

#### 시나리오 A: "긴급 민원 5분 내 초안 작성"
```
목표: 공무원이 출근 직후 긴급 민원을 받았을 때, 
      5분 내에 검토 가능한 초안을 받는다.

현재 문제점 파악 → 병목 제거 → 자동화 증대
```

#### 시나리오 B: "환각 탐지로 결재 반려 방지"
```
목표: AI가 생성한 문서에서 거짓 정보를 99% 잡아낸다.

패턴 규칙 확대 → LLM 검증 강화 → UI에서 명확히 표시
```

#### 시나리오 C: "비전문가도 5초 만에 시작"
```
목표: 법령 지식 없는 신입도 헷갈리지 않고 사용.

온보딩 개선 → 예시 버튼 → 실수 방지 가드레일
```

## 🔧 구체적 개선 요청

   - Lazy loading 적용

1. **후속 질문 UI (5152번째 줄)**
   - 자동완성 제안 추가

2. **검증 로그 UI (`render_verification_log` 함수)**
   - 아코디언 → 타임라인 시각화
   - 각 단계별 소요 시간 표시

3. **로딩 애니메이션**
   - 단조로운 스피너 → 단계별 애니메이션

2. **설정 관리 개선**
   - secrets.toml 없어도 데모 모드로 작동
   - 환경 변수 우선순위 체계
   
3. **로깅 체계 구축**
   - 성능 병목 자동 탐지
   - 사용자 행동 패턴 분석
